Why FOSS?

FOSS stands for Free and Open Source Software


If you use a lot of programs on your computer, you most likely have something FOSS. When using Linux, you are primarily running FOSS. Use OBS? That's FOSS. In contrast to proprietary software, the source code will be available for anyone to view and modify (depending on the license). The Free in FOSS refers to the actual freedom of the software, not the price (although it is usually free to use). There are many advantages to choose this over proprietary software, but of course downsides as well.


Privacy

FOSS will usually not have telemetry built in or enabled by default. It allows the software to be much more trustworthy for end users, while still receiving feedback from bug reports, PRs, suggestions, etc. Unfortunately, there are still bad apples. Some time ago, Audacity contributors had proposed adding telemetry into it. They received some heavy backlash on top of some other controversies relating to license policies. This is also what caused the creation of Tenacity, a fork of Audacity that doesn't have all the sussy stuff going on. you should use this over audacity since it also has some neat additions


Forks

Extending on the last topic, forks are a very significant part of FOSS. If something is to stop being maintained, someone can easily fork the project and continue it. If a project has bad actors, there's most likely a fork that exists. It allows many projects to continue thriving even though it's original core may be in shambles. Forks also allow developers to make their own modifications to the project without needing to gain approval from the original maintainer.


Contributions

Open source licenses allow people to freely fork and modify existing codebases. If one chooses to, they can open a pull/merge request in order for the modifications to be put into the original repo. It can range from translations, code, art, corrections, etc. Everything is maintained by the community, which makes it much easier to be able to directly communicate with the maintainer(s), as long as they are around.


Safe (mostly)

Most of the repos you see on GitHub (or any other repo hosting service) will be safe. No malware, no bloatware, no sussy stuff (mostly). You can't really get away with putting malicious stuff into a project for very long. Everything is documented via commits, which show exactly what has been modified since the last change. That doesn't mean there won't be a project that contains something malicious at some point, whether in or out of their control. A good way of filtering some of the more sussy projects is look at projects that have a sizable amount of stars, e.g. putting stars:<100 into your search query on GitHub. Of course, the best way to tell if something is safe or not just manually reviewing the source code, but not everyone knows how or is willing to do that.


Free (mostly)

Free as in price. You shouldn't really be paying for proprietary software if it's sussy and expensive. There are many great FOSS alternatives to proprietary software, but that's a different topic. Of course, there's no way around paying for an objectively good piece of software (well..) but using a quality alternative is still better than going in debt and having your stuff be used for ai.


Just useful.

Many tools are just available on GitHub, but may not appear on the front page of search engines. Some obscure issue you have could probably be solved by an open source project that you never found out about when you looked for a solution.
